What is event based programming?

First some history...

Traditionally, scalability has been achieved by using multiple threads or multiple processes. Apache for instance, can be configured to use either.

Since threads can run concurrently in a single process, a well written threaded program can be extremely fast.

Unfortunately Amdahl's law states (basically) that there is always some part of a system that can't be parallelized and therefore infinite cores does not yield infinite speedup.

But mostly threads suck because threaded programming is hard.

What is event based programming?

event
(noun)

Event based programs are based on a loop that sits around and waits for... events. An event can be a timer, network packet, mouse click, basically anything that the operating system can notify you about asynchronously.

A function that registers itself to be notified on a given event is named a callback and that callback handles the event.
